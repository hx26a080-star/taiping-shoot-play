# 画面遷移（シーン）仕様書

ゲームは主に以下の4つの画面（シーン）で構成され、それぞれの条件に応じて遷移します。

## 1. 画面（シーン）一覧

| 画面名 | 概要・表示内容 |
| :--- | :--- |
| **タイトル画面** | ゲーム起動時に表示される画面。ステージ選択へ進むボタンを配置。 |
| **ステージ選択画面** | プレイするステージ（1〜5）を選択する画面。 |
| **ステージ画面** | 実際に2Dシューティング（タイピング）を行うゲームメイン画面。 |
| **クリア画面** | ステージの挑戦結果（成功 / 失敗）と次のアクションへのナビゲーションを表示するリザルト画面。 |

---

## 2. 画面遷移フロー

```
[タイトル画面]
      │
      ▼ (ステージ選択へ進むボタン)
[ステージ選択画面]
      │
      ▼ (ステージ選択)
[ステージ画面]
      │
      ├─► (時間内に全撃破：成功) ──► [クリア画面 (成功表示)]
      │                                   ├─►「ステージ選択に戻る」──► [ステージ選択画面]
      │                                   ├─►「もう一度挑戦する」──► [ステージ画面 (同ステージ)]
      │                                   └─►「次のステージへ」  ──► [ステージ画面 (次ステージ)] ※5ステージ目クリア時は非表示
      │
      └─► (タイムアップ：失敗)   ──► [クリア画面 (失敗表示)]
                                          ├─►「ステージ選択に戻る」──► [ステージ選択画面]
                                          └─►「もう一度挑戦する」──► [ステージ画面 (同ステージ)]
```

### 2.1. 各画面の遷移アクション
*   **タイトル画面**
    *   「ステージ選択へ」進むボタンを押すと、**ステージ選択画面**へ遷移。
*   **ステージ選択画面**
    *   ステージ1〜5のいずれかを選択（クリック）すると、選択した**ステージ画面**へ遷移。
*   **ステージ画面**
    *   制限時間内にすべての敵を撃破すると、成功として**クリア画面**へ遷移。
    *   制限時間内に撃破できなかった場合、失敗として**クリア画面**へ遷移。
    *   **ライフ喪失による失敗**：ミス入力が5回に達した場合、強制的に失敗として**クリア画面**へ遷移。
    *   **リタイアボタン**：画面上部に配置。押下すると即座にプレイを中断し、失敗として**クリア画面**へ遷移。
*   **クリア画面**
    *   **成功時（クリア時）**の表示：
        *   「成功」というテキストを表示。
        *   **リザルト表示項目**：
            *   **クリアタイム**（クリアまでにかかった秒数、小数第2位まで表示）
            *   **撃破数**
            *   **最大コンボ数**（そのプレイ中に途切れずに連続して倒した最大数）
            *   **タイピングの正確さ**：ミスタイプ数（ターゲットへの入力ミス回数）
        *   アクションボタン：
            1.  「ステージ選択に戻る」
            2.  「もう一度挑戦する」
            3.  「次のステージへ」（※ただし、最終ステージであるステージ5のクリア時は非表示、または選択不可とする）
    *   **失敗時（ゲームオーバー時）**の表示：
        *   「失敗」というテキストを表示。
        *   **リザルト表示項目**：
            *   **経過時間**（タイムアップのため「60.00秒」または制限時間）
            *   **撃破数**
            *   **最大コンボ数**（そのプレイ中に途切れずに連続して倒した最大数）
            *   **タイピングの正確さ**：ミスタイプ数（ターゲットへの入力ミス回数）
        *   アクションボタン：
            1.  「ステージ選択に戻る」
            2.  「もう一度挑戦する」
            （「次のステージへ」ボタンは非表示）

## 3. 音響効果仕様
ゲーム体験向上のため、以下のタイミングで効果音（SE）を再生します。外部の音声ファイルを使用せず、ブラウザのWeb Audio API（シンセサイザー機能）を用いた合成音で再生します。

*   **キー入力成功時**：高めのピッチで小気味よく鳴るサウンド。
*   **ステージクリア時（成功時）**：明るく上昇するようなクリアファンファーレサウンド。
*   **タイムアップ時（失敗時）**：低く下降するようなゲームオーバーサウンド。

### 3.1. 効果音の画面仮表示（デバッグ・視覚フィードバック表示）
効果音が再生された瞬間、実感を得やすくするために、画面上（例えば画面下部やサイドの通知エリア）に「仮の視覚的インジケータ」を一定時間（1〜2秒程度）表示します。
*   **「キー入力成功時」再生時**：画面に「🎵 [SE: Key Success]」と仮表示。
*   **「ステージクリア時」再生時**：画面に「🎉 [SE: Stage Clear]」と仮表示。
*   **「タイムアップ時」再生時**：画面に「⚠️ [SE: Stage Failed]」と仮表示。
